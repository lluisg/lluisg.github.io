Exploring Lifelong Learning in Neural Machine Translation

January - July 2021

Description:

Implementation of the first lifelong learning system on Neural Machine Translation with the objective to suggest and analyze the usage of an active learning technique: Quality Estimation (QE), and the following comparison with the results obtained using random selection, and the subsequent study and analysis of the results obtained. In this project, we worked with the language pairs of English-French and English-German with the Active Learning techniques using the BEAT platform. For the projects we used two sets of data: one for training and the other to apply the learning without the translations, having a simulated person to whom we can request the translations for the sentences we think are necessary. As results with QE we achieve a score of 26.7 and 15.9 points for EN-FR and EN-DE, respectively, using a penalizing n-grams precisions BLEU score. These means an improvement of 0,5 for EN- FR and 0,7 for EN-DE over the results obtained using random selection.

Python

Youtube Views Predictor

October 2020 - February 2021

Description: (version 27/07/2021)

Youtube is one of the most important platforms in the world, and there exists a lot of interest in the content creators side to predict the number of views their future video will get. As such I created an Deep Learning architecture using an LSTM model to try to predict the future views obtained by a video on a channel taking into account the information available from their previous uploaded videos. During this I found that not much suitable data could be found on the internet, so a recompiled sequential information of the videos uploaded by different channels on Youtube. To obtain the data I wanted from Youtube, I used the data from 3 different sources: Trending YouTube Video Statistics by Mitchell J, Youtube Video and Channel Metadata by Vishwanath Seshagiri and the YouTube-8M Dataset. I started collecting data around November 2020, although there have been some hiatus. Since then, I have compiled more than 171.000 unique channels from the original 3 sources. From those channels, I collected information of more than 388.000 unique videos from more than 19.800 channels. And with those videos information I built a model that obtained predictions with 21.38% accuracy.

Transformer Implementation Project

Description:

I tried to implement a Transformer model, the state of the art for NLP and many other tasks, in order to learn better how it works. To do so, I designed a simple task such as to fill the blank word in a sentence (the language selected was Spanish). Jumbled Letters Project

August 2020

Description:

"Aoccdrnig to a rscheearch at Cmabrigde Uinervtisy, it deosn't mttaer in waht oredr the ltteers in a wrod are, the olny iprmoetnt tihng is taht the frist and lsat ltteer be at the rghit pclae. The rset can be a toatl mses and you can sitll raed it wouthit porbelm. Tihs is bcuseae the huamn mnid deos not raed ervey lteter by istlef, but the wrod as a wlohe." Since 2003, this sentence has been circulating on the internet. However, it seems that there never was a Cambridge research about it, but the general public has been debating for some time on the reason behind why we can read that particular jumbled text. Starting from this premise, I created and deployed model using Pytorch, SageMaker and a Jupyter Notebook that tries to reconstruct the original sentence from the jumbled letter of the words, maintaining the same first and last letter of each word for the final project of the Machine Learning Engineer Nanodegree at Udacity. For the data I used a subset of 1,499 blogs out of 19,320 on the Blogger Corpus. This model consisted of an LSTM with an encoder-decoder architecture, with a Fully Connected Layer connected to the output. It achieved a performance of 30% of fully correct reconstructed words and 63% of right guessed letters from 20,000 words. An additional simple HTML file was created to implement it on the web.

Culinary Chatbot

September 2018 - February 2019

Description:

We created a culinary chatbot for telegram as part of the Advanced Engineering Project. This chatbot, from a set of ingredients (sent to it in form of text or images), recommends what recipes can be done with them. At the beginning we got an initial version of the chatbot from the previous project. This version had implemented some chatbot functionality and made calls to two different REST API's. Spoonacular, to get recipes from a set of ingredients and Amazon Rekognition, to label the images sent by the users. The biggest differentiations from the previous version are: 1. Be independent from any external REST API. That meant, we had to implement our own neural network to recognize ingredients and also, in order to be independent from Spoonacular, we had to model a Recipe, get the data, save it as SQL and implement the full API for this database in order to be able to use it. 2. Create a user profile where we can have information about individuals. With this we accomplished to know better our customer and based on his diet, recommend only recipes that fit him. We also added a ranking system by which the recipes could get an overall ranking and the users could save recipes as 'favorite'. To recommend a recipe by its ingredients, we implemented an algorithm that classified the recipes by its ingredients, measuring how well those ingredients 'fitted' with the searched ones. We also implemented a histogram based algorithm to recommend a recipe knowing which were the last recipes the user had searched and which rank he had rated them with. The chatbot could be used from december 2018 to february 2019 at @culinary_chatbot on Telegram
